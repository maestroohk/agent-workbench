# firstmate

A multi-agent command orchestrator. The same author as `no-mistakes`,
`treehouse`, and `gnhf`.

## Purpose

firstmate is a **directory + `AGENTS.md` harness** (cloned to
`~/firstmate`), not a CLI binary with `firstmate doctor` /
`firstmate test` / `firstmate build` subcommands. Those names appear
in some third-party guides but they do not exist upstream.

The workbench uses firstmate to fill the `orchestrator` role — the
top-level coordination layer for multi-agent work. See `tools/roles.md`
for the role taxonomy and `tools/kunchenguid.md` for how the related
kunchenguid tools (`no-mistakes`, `treehouse`, `gnhf`, `lavish-axi`)
fit in.

## What firstmate ships

- `AGENTS.md` — the operating manual a Claude/Codex/OpenCode session
  loads when launched inside the harness. Tells the model how to
  route work to crewmates and read the harness's own state.
- `bin/fm-*.sh` — the toolbelt of verb scripts. The shim we install
  at `~/.local/bin/firstmate` (and `firstmate.cmd` on Windows)
  dispatches `firstmate <verb>` to `fm-<verb>.sh`. A bare `firstmate`
  call runs `fm-bootstrap.sh`. Unknown verbs return a list of the
  available scripts and exit 64.
- `bin/firstmate-shim.sh` — the dispatcher itself, generated by the
  workbench's `agent-init --bootstrap=firstmate`. It is rewritten on
  every reinstall so a corrupted shim is self-healing.

## Where it provides value in an AI workflow

- The orchestrator role: instead of a single Claude in a single
  checkout, firstmate splits a request into crewmates, each in its
  own worktree, and reconciles their results. See `tools/roles.md`
  for the role it fills.
- For a one-shot multi-agent dispatch, the workbench offers
  `agent-fleet N` (herdr or treehouse backend) which is the
  recommended entry point. firstmate's crewmate pattern is the
  long-running equivalent — for a project you live in for weeks,
  not for a single session.

## Installation

firstmate is a directory + `AGENTS.md` harness, **not** a CLI binary.
Install by cloning the repo:

```bash
git clone https://github.com/kunchenguid/firstmate
```

The workbench's `agent-init --bootstrap=firstmate` (or `--bootstrap=all`)
does the clone for you and drops the `firstmate` shim into
`~/.local/bin/`. The probe checks (in order): a `firstmate` binary
on PATH, or `~/firstmate/AGENTS.md` existing on disk.

## Recommended usage

- The shim's job is to dispatch. `firstmate bootstrap` runs the
  harness's own bootstrap; `firstmate <verb>` runs the matching
  `fm-<verb>.sh`. Run `firstmate` (no args) or `firstmate bootstrap`
  to enter the harness's preflight.
- Commit `firstmate.toml` to a project if you want `agent-test` to
  shell out to a custom test runner. The format is supported by the
  workbench's `agent-test`; it is not a hard requirement (the
  workbench falls back to the project-native runner when no
  `firstmate.toml` is present).

A minimal config (optional):

```toml
[project]
name = "my-api"

[commands.test]
run = "dotnet test -c Release"
description = "Run the test suite"
```

## Integration with `agent-workbench`

- `firstmate` is in `DEFAULT_BOOTSTRAP_SET` — installed by
  `agent-init --bootstrap` (and `--bootstrap=all`) by default.
- `agent-check` reports the harness install path, the most recent
  commit, the shim resolution, and the count of `bin/fm-*.sh`
  scripts. It does **not** call `firstmate doctor` or
  `firstmate build` — those subcommands do not exist upstream.
- `agent-test` will use `firstmate test` if a `firstmate.toml` is
  present in the repo AND firstmate is detected. Otherwise it falls
  back to the project-native test runner (`pytest`, `dotnet test`,
  `npm test`, etc.). `agent-test --no-firstmate` skips the
  firstmate path.
- For one-shot multi-agent dispatch, prefer `agent-fleet N` (herdr
  or treehouse backend); firstmate's crewmate pattern is for
  long-running projects where the captain pattern fits the workflow.

## Limitations

- Another layer between the developer and the toolchain. If the
  toolchain's native CLI is good and stable, `firstmate` adds little.
- A wrong entry in `firstmate.toml` will fail every consumer — humans
  and agents alike. Keep the file under review.
- The upstream ships only `bin/fm-*.sh`; no `firstmate doctor` or
  `firstmate build`. Diagnostic output is in the scripts' own
  stdout/stderr and in the workbench's `agent-check` report.
